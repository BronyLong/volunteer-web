# **ЭКСПЛУАТАЦИОННАЯ ДОКУМЕНТАЦИЯ**
для веб-приложения волонтёрской организации

## **1. Общие сведения**

Веб-приложение предназначено для автоматизации деятельности волонтёрской организации и обеспечивает взаимодействие между волонтёрами, координаторами мероприятий и администраторами системы.

Система позволяет:

* регистрировать и аутентифицировать пользователей;
* управлять профилями пользователей;
* создавать, редактировать и удалять мероприятия;
* подавать и обрабатывать заявки на участие;
* контролировать доступ к данным в зависимости от роли пользователя;
* фиксировать действия пользователей в журнале аудита.

Приложение реализовано по клиент-серверной архитектуре:

* клиентская часть — React;
* серверная часть — Node.js + Express;
* база данных — PostgreSQL.

## **2. Требования к программному обеспечению**

Для эксплуатации системы требуется:

* Node.js;
* npm;
* PostgreSQL;
* веб-браузер (Google Chrome, Mozilla Firefox, Microsoft Edge).

## 3. Подготовка системы к работе**

### 3.1 Создание базы данных

1. Запустить PostgreSQL.
2. Создать базу данных:
```
CREATE DATABASE volunteer_org;
```
3.  Выполнить SQL-скрипт структуры:
```
psql -U postgres -d volunteer_org -f volunteer_org.sql
```
4. (При необходимости) Выполнить скрипт заполнения данных:
```
psql -U postgres -d volunteer_org -f data_inserts.sql
```
### 3.2 Настройка серверной части

1. Перейти в каталог сервера:
```
cd server
```
2. Установить зависимости:
```
npm install
```
3. Создать файл .env и задать параметры:
```
PORT=5000
DB_HOST=localhost
DB_PORT=5432
DB_NAME=volunteer_org
DB_USER=(ваш пользователь)
DB_PASSWORD=(ваш пароль)
CLIENT_URL=http://localhost:5173
JWT_SECRET=(ваш ключ)
```
### 3.3 Настройка клиентской части

1. Перейти в каталог frontend:
```
cd react-volunteer-app
```
2. Установить зависимости:
```
npm install
```
## 4. Запуск системы
### 4.1 Запуск backend

```
cd server
npm run dev
```

**Сервер запускается по адресу:

http://localhost:5000
### 4.2 Запуск frontend

```
cd react-volunteer-app
npm run dev
```

**Приложение доступно по адресу:

http://localhost:5173
## 5. Описание ролей пользователей
### 5.1 Гость (неавторизованный пользователь)

**Доступно:

- просмотр списка мероприятий;
- просмотр страницы мероприятия;
- регистрация;
- вход в систему.
### 5.2 Волонтёр

**Доступно:

- просмотр и редактирование профиля;
- просмотр мероприятий;
- подача заявки на мероприятие;
- отзыв заявки (до начала мероприятия или отмены заявки координатором);
- просмотр контактов координатора (после подачи заявки).

**Ограничения:

- нельзя подавать повторную заявку;
- нельзя подавать заявку на завершённое мероприятие.
### 5.3 Координатор

**Доступно:

- создание мероприятий;
- редактирование и удаление своих мероприятий;
- просмотр заявок на свои мероприятия;
- отклонение и восстановление заявок (до начала мероприятия).

**Ограничения:

- нельзя редактировать чужие мероприятия;
- нельзя изменять заявки после завершения мероприятия.
### 5.4 Администратор

**Доступно:

- полный доступ ко всем мероприятиям;
- просмотр всех заявок;
- просмотр профилей пользователей с полными данными;
- управление системой.
## 6. Руководство пользователя
### 6.1 Регистрация

1. Перейти на страницу регистрации.
2. Заполнить необходимые поля.
3. Нажать кнопку «Зарегистрироваться».

**Требования к паролю:

- не менее 8 символов;
- наличие заглавных и строчных букв;
- наличие цифр;
- наличие специальных символов;
- отсутствие пробелов.
### 6.2 Вход в систему

1. Перейти на страницу входа.
2. Ввести email и пароль.
3. Нажать «Войти».
### 6.3 Работа с профилем

1. Перейти в личный кабинет.
2. Нажать «Редактировать профиль».
3. Изменить необходимые данные.
4. Сохранить изменения.
### 6.4 Просмотр мероприятий

1. Перейти на страницу «Мероприятия».
2. Выбрать категорию (при необходимости).
3. Открыть карточку мероприятия.
### 6.5 Подача заявки

1. Открыть страницу мероприятия.
2. Нажать «Подать заявку».

**После подачи:

- отображается статус заявки;
- открываются контакты координатора.
### 6.6 Отзыв заявки

1. Перейти на страницу мероприятия.
2. Нажать «Отменить заявку».

**Доступно только до начала мероприятия.

### 6.7 Создание мероприятия (координатор)

1. Перейти на страницу создания.
2. Заполнить необходимые поля.
3. Опубликовать мероприятие.

### 6.7 Редактирование мероприятия (координатор)

1. Перейти на страницу мероприятия.
2. Нажать на кнопку "редактировать мероприятие"
3. Изменить необходимые поля.
4. Применить мероприятие.
### 6.8 Управление заявками (координатор)

1. Открыть своё мероприятие.
2. Перейти к списку заявок.
3. Выполнить действие:
	- отклонить;
	- восстановить.
## 7. Администрирование системы

**Администратор выполняет:

- контроль работоспособности сервера;
- проверку подключения к базе данных;
- управление данными пользователей;
- контроль журналов аудита.
## 8. Журналирование действий

**Система ведёт журнал действий пользователей, в котором фиксируются:

- регистрация;
- вход в систему;
- создание и редактирование мероприятий;
- подача и изменение заявок;
- редактирование профиля.

**Журнал хранится в таблице audit_logs.

## 9. Типовые ошибки и их устранение
### 9.1 Ошибка подключения к базе данных

**Причина:

- неверные параметры в .env.

**Решение:

- проверить DB_HOST, DB_PORT, DB_NAME, DB_USER, DB_PASSWORD.
### 9.2 Не отображаются данные

**Причина:

- backend не запущен.

**Решение:

- проверить запуск сервера.
### 9.3 Ошибка авторизации

**Причина:

- отсутствует или истёк JWT-токен.

**Решение:

- выполнить повторный вход.
### 9.4 Невозможно подать заявку

**Причина:

- нет свободных мест;
- заявка уже подана;
- заявка отклонена координатором
- мероприятие завершено.
