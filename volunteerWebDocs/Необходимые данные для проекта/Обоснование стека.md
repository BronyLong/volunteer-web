## Обоснование выбора технологического стека

Выбор технологического стека для разработки веб-приложения осуществлялся с учётом функциональных требований системы, предполагаемых сценариев использования и необходимости обеспечения удобного пользовательского интерфейса.

В качестве основных технологий были выбраны:

- React.js — для реализации клиентской части;
- Node.js и Express.js — для серверной части;
- PostgreSQL — для хранения данных;
- JWT — для аутентификации пользователей.

### Клиентская часть (React.js)

Для разработки пользовательского интерфейса выбрана библиотека React.js, которая широко используется для создания одностраничных приложений (SPA).

Использование React.js позволяет:

- реализовать динамический интерфейс без перезагрузки страниц;
- разбить интерфейс на независимые компоненты (например, карточка мероприятия, форма входа);
- обеспечить удобную навигацию между страницами;
- эффективно обновлять данные на странице.

Это особенно важно для реализации таких функций, как:
- просмотр списка мероприятий;
- отображение страницы мероприятия;
- подача заявки на участие;
- работа с профилем пользователя.

### Серверная часть (Node.js и Express.js)

Для реализации серверной логики выбран фреймворк Express.js, работающий на платформе Node.js.

Серверная часть обеспечивает:

- обработку HTTP-запросов от клиента;
- реализацию бизнес-логики системы;
- работу с пользователями и их ролями;
- управление мероприятиями;
- обработку заявок на участие.

Express позволяет удобно реализовать REST API и разделить логику по модулям.

### База данных (PostgreSQL)

В качестве системы управления базами данных выбрана PostgreSQL.

Использование PostgreSQL обеспечивает:

- надёжное хранение данных;
- поддержку связей между сущностями (пользователи, мероприятия, заявки);
- выполнение сложных запросов;
- обеспечение целостности данных.

### Аутентификация (JWT)

Для аутентификации пользователей используется технология JSON Web Token (JWT).

Механизм работы:

1. Пользователь вводит email и пароль;
2. сервер проверяет данные;
3. при успешной авторизации выдает токен;
4. токен используется для доступа к защищённым маршрутам.

Преимущества JWT:

- отсутствие необходимости хранения сессий;
- удобная интеграция с REST API;
- повышение безопасности системы.

## Итог

Выбранный технологический стек обеспечивает:

- удобную разработку клиентской и серверной части;
- поддержку всех необходимых функций системы;
- возможность дальнейшего расширения приложения;
- соответствие современным требованиям к веб-приложениям.
