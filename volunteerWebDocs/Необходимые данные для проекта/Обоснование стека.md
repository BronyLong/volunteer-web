Выбор технологического стека для разработки программной системы осуществлялся на основе анализа функциональных требований к разрабатываемому приложению, предполагаемых сценариев использования, требований к производительности, масштабируемости, безопасности, а также с учетом опыта использования данных технологий в процессе обучения. Особое внимание уделялось поддержке основных бизнес-процессов волонтёрской организации: регистрации и аутентификации пользователей с разными ролями, работе с мероприятиями, учёту волонтёрских часов и предоставлению удобного веб-интерфейса.

В ходе учебного процесса данные технологии уже изучались и применялись при выполнении лабораторных и практических работ, что позволило сформировать необходимый уровень компетенций для их эффективного применения в рамках данного проекта. Это, в свою очередь, позволяет сократить время разработки, снизить вероятность ошибок при реализации и обеспечить более качественную реализацию программного продукта.

В качестве основных технологий были выбраны: библиотека React.js для реализации клиентской части приложения, серверный фреймворк Express.js, функционирующий на платформе Node.js, система управления базами данных PostgreSQL, а также механизм аутентификации пользователей на основе стандарта JSON Web Token. Ниже приведено подробное обоснование выбора каждой из указанных технологий.
### Клиентская часть на React.js 

Для разработки пользовательского интерфейса веб-приложения была выбрана библиотека React.js, которая является одним из наиболее распространенных инструментов для создания современных веб-интерфейсов. React.js активно применяется при разработке одностраничных приложений (SPA – Single Page Application), где значительная часть логики взаимодействия с пользователем выполняется на стороне клиента. Такой подход хорошо соответствует идее проекта: пользователь после авторизации получает доступ к личному кабинету и разделам системы без полной перезагрузки страниц.

Одним из ключевых преимуществ React.js является компонентный подход к построению интерфейса. В рамках данного подхода пользовательский интерфейс разбивается на независимые компоненты, каждый из которых реализует отдельный функциональный элемент интерфейса (например, карточка мероприятия, форма входа, виджет статистики волонтёрских часов). Это позволяет значительно повысить модульность программного кода, облегчает его сопровождение и повторное использование компонентов в различных частях приложения.

React.js использует механизм виртуального DOM, который представляет собой внутреннюю структуру, отражающую текущее состояние пользовательского интерфейса. При изменении данных React сначала обновляет виртуальное представление интерфейса, после чего вычисляет минимальный набор изменений, которые необходимо внести в реальный DOM браузера. Такой подход существенно снижает количество операций перерисовки страницы и позволяет повысить производительность приложения, что особенно актуально при работе со списками мероприятий, статистическими данными и другими динамическими разделами.

Дополнительным преимуществом React является наличие развитой экосистемы инструментов и библиотек. В частности, доступны библиотеки для маршрутизации, управления состоянием приложения, обработки HTTP-запросов, работы с формами и валидацией данных. В рамках данного проекта предполагается использование специализированных средств для отправки HTTP-запросов к серверному API и организации навигации между основными разделами (главная страница, список мероприятий, личный кабинет и др.). Наличие большого количества готовых решений позволяет ускорить разработку интерфейса и снизить объём кода, который необходимо реализовывать вручную.

Использование React.js в рамках данного проекта обеспечивает:
- высокую скорость разработки пользовательского интерфейса;
- модульную структуру клиентского приложения;
- высокую производительность при работе с динамическими интерфейсами;
- удобство масштабирования и сопровождения программного кода;
- возможность реализации ролевой навигации и разграничения доступа на уровне клиентского интерфейса.
### Серверная часть на Express 

Для реализации серверной части приложения был выбран веб-фреймворк Express, работающий на платформе Node.js. Данный фреймворк предназначен для создания веб-приложений и REST-ориентированных API и широко используется при разработке серверной логики современных веб-систем. Использование Express и Node.js позволяет эффективно обрабатывать большое количество сравнительно коротких HTTP-запросов, характерных для информационных систем с активной работой с данными (просмотр списков, подача заявок, подтверждение часов и т.д.).

Express характеризуется минималистичной архитектурой, предоставляя разработчику базовый набор инструментов для обработки HTTP-запросов, организации маршрутизации и использования промежуточных обработчиков (middleware). Такой подход позволяет гибко формировать архитектуру серверного приложения и адаптировать её под требования конкретного проекта, разделяя маршруты, связанные с пользователями, мероприятиями, заявками и отчётностью.

В рамках разрабатываемой системы серверная часть выполняет следующие функции:
- обработку запросов, поступающих от клиентского приложения;
- реализацию бизнес-логики системы (регистрация и аутентификация пользователей, работа с ролями и правами доступа, регистрация волонтёров на мероприятия, учёт и подтверждение волонтёрских часов);
- взаимодействие с базой данных;
- проверку подлинности пользователей;
- формирование ответов клиентскому приложению в формате JSON.

Использование Express позволяет удобно реализовать REST-архитектуру взаимодействия между клиентской и серверной частью приложения. Клиентское приложение отправляет HTTP-запросы к серверу, сервер обрабатывает их, выполняет необходимые операции и возвращает результат обработки.

Ещё одной важной особенностью Express является поддержка механизма middleware. Middleware представляет собой промежуточные функции обработки запроса, которые выполняются последовательно. С их помощью можно реализовать различные служебные функции, такие как:

- логирование запросов;
- обработка ошибок;
- проверка аутентификации пользователей;
- валидация входных данных.

Использование Express в рамках данного проекта обеспечивает:

- простоту разработки серверной части;
- высокую скорость обработки HTTP-запросов;
- удобную реализацию REST API;
- гибкость архитектуры серверного приложения.

### СУБД PostgreSQL и аутентификация на основе JWT

Для хранения данных системы была выбрана реляционная система управления базами данных PostgreSQL. Данная СУБД является одной из наиболее распространенных систем управления базами данных с открытым исходным кодом и широко применяется в промышленной разработке программных систем.

PostgreSQL поддерживает стандарт языка SQL и обладает широким набором возможностей для работы с данными, включая:

- поддержку сложных запросов;
- использование индексов для оптимизации доступа к данным;
- работу с транзакциями;
- использование представлений, триггеров и хранимых процедур.

Одним из ключевых преимуществ PostgreSQL является поддержка ACID-свойств транзакций, обеспечивающих надежность и целостность данных при выполнении операций с базой данных. Это особенно важно для информационных систем, в которых выполняются операции добавления, изменения и удаления данных.

Кроме того, PostgreSQL обеспечивает:

- высокую надежность хранения данных;
- возможность масштабирования системы;
- эффективную работу с большими объемами информации;
- активную поддержку со стороны сообщества разработчиков.

Для реализации механизма аутентификации пользователей используется технология JWT (JSON Web Token). JWT представляет собой стандарт передачи информации между сторонами в виде защищенного токена, подписанного криптографическим алгоритмом.

Процесс аутентификации с использованием JWT включает следующие этапы:

1. Пользователь вводит учетные данные (логин и пароль).
2. Сервер проверяет корректность введенных данных.
3. В случае успешной проверки сервер формирует токен JWT, содержащий информацию о пользователе.
4. Токен передается клиентскому приложению.
5. При выполнении последующих запросов клиент отправляет данный токен в заголовке HTTP-запроса.
6. Сервер проверяет подпись токена и предоставляет доступ к защищенным ресурсам системы.

Основными преимуществами использования JWT являются:

- отсутствие необходимости хранения пользовательских сессий на сервере;
- удобство интеграции с REST-архитектурой;
- возможность масштабирования системы;
- повышение безопасности взаимодействия клиента и сервера.
