# ОПРЕДЕЛЕНИЯ

**Веб‑приложение** — программная система, доступная пользователям через сеть Интернет при помощи веб‑браузера.
**API (Application Programming Interface)** — интерфейс программирования приложений, обеспечивающий взаимодействие между программными компонентами.
**Frontend** — клиентская часть веб‑приложения, отвечающая за пользовательский интерфейс.
**Backend** — серверная часть приложения, реализующая бизнес‑логику системы.
**База данных** — структурированное хранилище информации, используемое для долговременного хранения данных системы.
**JWT (JSON Web Token)** — технология передачи данных между сторонами в виде подписанного токена, используемая для аутентификации пользователей.
**Волонтёр** — пользователь системы, принимающий участие в мероприятиях.
**Координатор** — пользователь, организующий мероприятия и управляющий участниками.
**Администратор** — пользователь, обладающий расширенными правами управления системой.

# 2. НАЗНАЧЕНИЕ И ЦЕЛИ СОЗДАНИЯ СИСТЕМЫ
## 2.1 Назначение системы
Разрабатываемая система предназначена для автоматизации деятельности волонтёрской организации и управления информацией о:
- пользователях системы;
- волонтёрских мероприятиях;
- заявках на участие;
- отработанных волонтёрских часах;
- статистике активности участников.
Система обеспечивает централизованное хранение данных и упрощает взаимодействие между участниками волонтёрской деятельности.
## 2.2 Цели создания системы
Основными целями разработки системы являются:
1. Автоматизация процессов управления волонтёрами.
2. Повышение эффективности планирования мероприятий.
3. Создание централизованной базы данных волонтёрской активности.
4. Упрощение взаимодействия между волонтёрами и координаторами.
5. Повышение прозрачности деятельности волонтёрской организации.
6. Создание удобной онлайн‑платформы для привлечения новых волонтёров.
# 3. ХАРАКТЕРИСТИКИ ОБЪЕКТА АВТОМАТИЗАЦИИ
Объектом автоматизации являются процессы деятельности волонтёрской организации, включая:
- регистрацию и управление пользователями;
- создание и управление мероприятиями;
- регистрацию волонтёров на мероприятия;
- учет волонтёрских часов;
- администрирование системы.
В системе предусмотрены следующие роли пользователей:
- гость;
- волонтёр;
- координатор;
- администратор.
Каждая роль обладает собственным набором прав доступа и функциональных возможностей.
# 4. ТРЕБОВАНИЯ К ПРОГРАММНОМУ ПРОДУКТУ
## 4.1 Требования к системе в целом
Система должна иметь многослойную архитектуру, включающую:
- клиентский уровень (frontend);
- серверный уровень (backend);
- уровень хранения данных (database).
Такое разделение обеспечивает:
- масштабируемость системы;
- удобство сопровождения;
- разделение ответственности между компонентами.
## 4.2 Функциональные требования
### 4.2.1 Возможности для гостей
Гость системы должен иметь возможность:
- просматривать главную страницу;
- просматривать список мероприятий;
- просматривать подробную информацию о мероприятиях;
- зарегистрироваться в системе.
### 4.2.2 Возможности для волонтёров
Волонтёр должен иметь возможность:
- управлять своим профилем;
- просматривать список мероприятий;
- подавать заявки на участие;
- отменять заявку до подтверждения;
- просматривать статус заявок;
- просматривать историю участия;
- просматривать статистику волонтёрских часов.
### 4.2.3 Возможности для координаторов
Координатор должен иметь возможность:
- создавать мероприятия;
- редактировать мероприятия;
- управлять заявками волонтёров;
- подтверждать участие волонтёров;
- подтверждать волонтёрские часы;
- просматривать статистику по мероприятиям.
### 4.2.4 Возможности для администраторов
Администратор должен иметь возможность:
- управлять пользователями;
- назначать роли пользователей;
- блокировать и разблокировать аккаунты;
- управлять мероприятиями;
- просматривать статистику системы.
# 5. НЕФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ
## 5.1 Производительность

- интуитивно понятным;
- адаптивным;
- доступным для пользователей без специальной подготовки.
## 5.3 Безопасность

- аутентификацию пользователей;
- хранение паролей в виде хешей;
- разграничение доступа на основе ролей.
